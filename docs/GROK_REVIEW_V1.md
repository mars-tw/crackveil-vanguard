## (1) 合成層時序：大片還是 PPT？

### 1.1 各層職責（結構評分：**大片向，B+**）

爆炸（`explosion_area.gd` `_update_sprite_state`）在 **同一 normalized time `t = age/lifetime`** 上疊四種角色：

| 層 | 登場 | Scale 行為 | Alpha 行為 | 電影角色 |
|----|------|------------|------------|----------|
| **Core flash** | t=0 起，`flash_t=t/0.24` | 1.15→2.4 線性 | (1−flash_t)×0.96 | 核閃 — 應最短、最亮 |
| **主爆形** | t=0 | 半徑×(2.25+0.42t) 緩脹 | `1−smoothstep(0.28,1)` 前 28% 滿透再淡 | 火球 silhouette |
| **衝擊環** | t=0，`ring_t=t/0.82` | 1.2→3.65 **線性** | (1−ring_t)×0.82 **線性** | 壓力波 |
| **Smoke** | **t≥0.18** 延遲 | 1.35→3.05 線性 | `sin(smoke_t·π)×0.44` 鐘形 | 殘煙 |
| **Debris** | 僅 layer≥4，one-shot | 粒子速度∝半徑 | 固定 0.88 | 碎片 |

Death burst（`death_burst.gd`）同構：smoke 延遲 **0.16**；ring 完成於 **0.76**（`boss_phase` 0.92）；preset 把 ring 拉到 146→230→340→**720**。

投射物（`projectile.gd`）不走四層爆炸，而是：

- Muzzle：`0.06s` 線性淡出（短 pop，合理）  
- Trail：3 點 `Line2D` 靜態長度（速度×時間 clamp），非粒子拖尾  
- Lob：`sin(t·π)` 抬升（弧線正確）  
- Homing 命中：雙 burst（ember + smoke_ring）— **有分層意識**  
- Boss ring：sprite 自旋 + glow sin 脈動  

**結構結論**：**成立** — 「核閃 → 主體 → 環 → 煙」是工業標準爆炸分鏡，不是隨機疊圖。

### 1.2 時序曲線：哪裡像大片、哪裡像 PPT？

#### 已具專業感（保留）

1. **煙延遲登場**（爆炸 18%／死亡 16% 壽命）— 正確。真實煙在壓力波之後。  
2. **煙 alpha 用 sin 半波** — 淡入再淡出，比線性砍刀好一截。  
3. **主形 alpha 用 smoothstep(0.28→1)** — 前段「撐住火球」、後半才溶掉，比全程線性專業。  
4. **Core flash 壓縮在前 24%**（Boss 34%）— 短閃正確。  
5. **Death preset 壽命階梯**（spark 0.24 → burst 0.32 → elite 0.58 → boss_phase **0.92**）— 高光時刻給足閱讀時間。  
6. **Boss 環 720 + 獨立 Line2D shockwave** — 戲劇尺度對。

#### PPT 感來源（需點名）

| 問題 | 碼象 | 觀感 | 嚴重度 |
|------|------|------|--------|
| **衝擊環 scale 線性** | `lerpf(1.2, 3.65, ring_t)`；death 同 | 等速膨脹＝PowerPoint 圓環動畫；大片應 **先快後慢 ease-out**（空氣阻力感） | **P1** |
| **環 alpha 線性** | `(1−ring_t)×k` | 可接受，但若 scale 已 ease-out，alpha 宜 **中段仍亮、末端急收** | P2 |
| **全層共用 lifetime 收場** | 爆炸預設 **0.32s** 全層一起死 | 煙在 t=1 被硬切；電影應讓煙 **殘留 1.3–1.6×** 主壽命 | **P1** |
| **Glow 線性 (1−t)** | `glow.modulate.a = (1−t)×0.5` | 身份柔光同步「關燈」，無餘韻 | P2 |
| **Death 主 sprite 等速旋轉** | `sprite.rotation += 1.8 * dt` | 小玩具感；精英／Boss 可用極慢或停轉 | P2 |
| **Boss Line2D 環亦線性** | `lerpf(18, max_radius, t)` | 全場波最需要 ease-out，卻最線性 | **P1**（高光時刻） |
| **投射物 trail 靜態折線** | 3 點固定 | 可讀、不髒；截圖靜態 OK，動圖略「幾何」 | P2 |

#### 衝擊環「專業感」公式對照（審查建議，不實作）

理想（概念）：

```
ease_out = 1 - (1 - ring_t)^2.5   # 或 ^3
scale   = lerpf(1.15, 3.8, ease_out)
alpha   = (1 - ring_t^1.6) * 0.85
```

現況：`ease = identity`。  
**觀測判準**：若逐幀截圖環半徑近似等差數列 → PPT；若前期幀距大、後期幀距小 → 大片。

#### Smoke「延遲＋緩淡」現況評分

| 檢查項 | 期望 | 現況 | 判定 |
|--------|------|------|------|
| 延遲登場 | 有 | 0.16–0.18 | **成立** |
| 緩慢淡出 | 尾段長 | sin 對稱，後半與前半等長；且 **t=1 與主體同死** | **部分成立** |
| 色溫 | 冷灰／褐 | 固定 `(0.52,0.46,0.5)`／death `(0.48,0.43,0.52)`，幾乎不跟武器色 | 可讀，略「貼紙」 |
| 與 ember／cyan 色盤 | 跟隨 | 紋理 suffix 跟 `_uses_ember_palette()`，modulate 不跟 | 部分成立 |

**§1 總評**：  
- **分層敘事 = 大片草稿**  
- **easing 與壽命編排 = 仍偏簡報**  
- 單一參數改動（ring ease-out + smoke 獨立 lifetime）即可跨過「Steam GIF 耐看」門檻，無需新貼圖。

```

inner: scale 0.82→1.12 跟 pulse；alpha 0.34→0.64 跟 pulse
```

## (7) 結論

| 命題 | 裁決 |
|------|------|
| 四層合成「像大片」 | **結構成立、曲線未完成** → 半大片半 PPT |
| 煙延遲登場 | **成立**；緩慢淡出 **部分成立**（缺跨壽命殘留） |
| 衝擊環先快後慢 | **未達**（線性） |
| 11 武器色與主題／敵彈 | **主題對齊**；**冰青簇＋玩家 ember vs 敵彈** 為主要風險 |
| Boss 體積光有層次 | **意圖成立、完成度中等** |
| 可進 Steam？ | **可**，條件是選對鏡頭並優先做 §4 順位 1–3；**不宜**用滿場 LOD=2 當主視覺 |

**監工簽字意見**：**只審不改完成**。下一輪視覺若只准做三件事：**(1) ring ease-out + smoke linger (2) 冰青形差 / 敵彈第二通道 (3) Boss 光 alpha 階梯** — 預期 Steam 就緒度可從 ~6.5 推到 ~8.0，且不必然觸碰 M4 效能紅線。
