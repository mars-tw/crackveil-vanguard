# Crackveil Vanguard Press Kit

## R12 最新特色

- Seed 決定三種地圖主題：裂隙虛空、廢土農野、餘燼裂原。
- 玩家小隊新增程序步伐、小跳步、步塵與低音量腳步 tick，敵人使用輕量步相位。
- 地圖新增 decor 散佈、裂隙脈衝、遠景裂隙閃電、植物微晃與偶發流星/碎片。

## 30 字 Pitch

五人裂隙小隊割草生存：契約改規則、詞綴精英破節奏、武器進化與殘響 Meta 推你再開一局。

## 線上網址

https://mars-tw.github.io/crackveil-vanguard/

## 操作

- `WASD` / 方向鍵：移動。
- `P` / `Esc`：暫停、設定、複製種子。
- 行動裝置：左下虛擬搖桿。
- 開局契約畫面：可輸入或貼上種子開新局。

## 特色

- 三支柱戰鬥：小隊站位、自動武器、敵潮壓力。
- 裂隙契約：開局選一條本局規則，改變風險與收益。
- 詞綴精英：裂殖、磁滯、疾閃三種規則怪，具備色彩與形狀提示。
- 武器進化：投入武器後解鎖質變卡，形成第二層 build。
- 殘響 Meta：結算帶回碎片，解鎖契約槽與小幅永久整備。
- 本地成就與種子分享：結算追蹤本局解鎖，玩家可複製同一局種子互相比拼。

## 瀏覽器需求

- Godot 4.7 單執行緒 WebAssembly build。
- 建議使用最新版 Chrome、Edge、Firefox 或 Safari。
- 不需要 SharedArrayBuffer；適合 GitHub Pages 等靜態站台。
- 首次播放音效需要一次點擊或按鍵以解鎖瀏覽器音訊。

## 截圖指南

### 拍攝前設定

- Desktop、16:9、至少 1280×720；不用 mobile LOD 拍主戰鬥圖。
- 主戰鬥圖先確認現場敵數約 60–100。敵數 ≥120 或 150 敵 Stress 只作效能證據，不作封面。
- Debug build 可按 `F12` 切換 `screenshot_beauty`：隱藏整個 HUD／虛擬搖桿，並讓切換後新生的爆炸與死亡效果固定保留 4 層。再按一次恢復自適應 LOD 與 HUD。亦可用 `--screenshot-beauty` 啟動；release build 不啟用此功能。
- Beauty 模式只協助取材，不改傷害、敵數、生成節奏或正式玩家的效能設定。

### 安全時刻窗（When）

依優先序取材：

1. Boss Phase 2 觸發後 0.5–2.0 秒：等紅熱核、危機橫幅與環彈出現；不要等到血條將盡狂閃。
2. 中密度交戰、約 55–95 敵：Desktop 仍保留 3 層煙與柔光；避開敵數 ≥120 或同時 ≥12 個 death burst。
3. 單發大爆炸／榴彈／脈衝著彈後、煙尾前 40%：主衝擊環已張開且 1.42× 煙尾仍亮。
4. 詞綴精英入場或被集火：保留放大的裂殖／磁滯／疾閃標記，不要等精英死光。
5. 升級卡滑入後 0.3–0.8 秒或結算 count-up 中段：只作產品／Meta 次圖，不作唯一主圖。

禁止把 150 敵 Stress、Boss／Tank 臉部極端 crop、英雄腳部特寫、開局前 20 秒空場、或全進化後一片 mint／白青當主視覺。

### 三鏡頭構圖（How）

- **L1 主宰對峙**：16:9；Boss 在右或上 1/3，小隊在左下，隊長冷青柔光可見。層級為 Boss 紅熱核 → 小隊剪影 → 1–2 種玩家武器色；五秒內應讀成「小隊正在打 Boss」。
- **L2 火力語彙**：中近景同框兩把形差最大的武器，首選 orbit 的 periwinkle 弧搭 rail 細白線或裂線長尾；可加一條榴彈 ember 弧。不要以 boomerang＋rail 兩條近白線作主圖。
- **L3 規則張力**：契約三選一、詞綴精英標記或進化卡三擇一；文案／符號必須可讀，讓觀者知道遊戲有 build 與規則，不只是割草。

### 構圖硬規則與故事自檢

- 同框保留「冷錨＋熱威脅」；深度至少有前景精英／掉落、中景小隊、遠景敵潮／地圖脈衝三帶。
- 主戰鬥圖 HUD 合計不超過畫面高度約 18%，並留 15–20% 虛空底或煙尾負空間。
- 不把彈幕沿對角線拉滿；保留一條清晰敵彈暗輪廓即可。武器用中景拍 trail 與爆炸，不拍角色臉。
- 每張圖至少帶一個空間錨：虛空遠景閃電、農野作物／農具剪影、或餘燼橙裂縫，約佔畫面 8–15%。
- 故事圖應同時回答：威脅是誰、我方在哪、哪條規則生效、身處哪張地圖。Boss 二階危機橫幅、精英 affix 標記、契約標題與地圖錨點各補一項。
