### 1.1 爆炸／死亡曲線 — 品質細讀

#### 做得對的地方

1. **公式照抄 V1 概念碼**，非「大概 ease」：  
   `ring_ease = 1 - (1 - ring_t)^2.5`、`alpha = (1 - ring_t^1.6) * k`。  
2. **death 與 explosion 同構**，Boss Line2D shockwave 也吃 ease-out — V1 點名的「全場波最線性」已關。  
3. **主體時間軸與煙壽命解耦**：  
   - `death_burst`：`t = age / main_lifetime`，釋放壽命 = `main * 1.42`（≥3 層）。  
   - `explosion_area`：`t = age / effect_lifetime`（**主壽命**），`_process` 用 `_visual_lifetime()` 決定回收 — **沒有**把主形曲線錯誤拉長到 1.42×。  
4. 煙 alpha 從對稱 `sin` 改為 `sin(smoke_t^0.82 · π) * (1-smoke_t)^0.28` — 尾段略拖、符合「殘留」意圖。  
5. 2 層 LOD **不**延壽 — 與 M4 效能契約一致，宣傳滿場仍乾、但不會偷偷拖長 burst 池佔用。

#### 品質保留意見（非阻擋）

| 項 | 嚴重度 | 說明 |
|----|--------|------|
| Glow 仍 `(1−t)` 線性關燈 | P2 | V1 已列、本輪未做；可接受 |
| Death 主 sprite 等速旋轉 | P2 | 未動；小玩具感殘留 |
| 煙色仍固定灰紫、不跟武器色 | P2 | 可讀優先；動圖略「貼紙」 |
| 迴歸只取兩段 scale 差分 | 測試債 | 能擋「退化回線性」，擋不住「ease 指數被改弱仍過」；夠用、非假綠 |

**§1.1 裁決**：**P1 第一刀達標** — 結構從「半 PPT」跨到「可當 Steam GIF 環」。

### 提升點 A — **安全時刻窗（When）**：什麼時刻最上鏡

目標：同一建置，**選對 2–4 秒** 就能穩定出「商店主圖級」靜幀／短 GIF，避開 LOD 乾場與糊焦點。

#### A.1 建議鎖定的「黃金窗」（優先序）

| 順位 | 時刻 | 為何上鏡 | 避開條件 |
|-----:|------|----------|----------|
| **1** | **Boss Phase 2 觸發後 0.5–2.0s** | 紅熱核 + 加速脈動 + `boss_phase` 4 層 burst + 環彈；V1 後最完整的「敵方主宰」語言 | 勿等血條將盡狂閃；勿在 150 敵 Stress 同框 |
| **2** | **中密度交戰：現場敵約 55–95** | Desktop **3 層**煙／柔光仍在（`<120`）；火力可讀、畫面不「像砍過的 build」 | 敵數 **≥120** 強制 2 層；live death_burst **≥12** 同砍 |
| **3** | **單發大爆炸／榴彈或脈衝著彈瞬間 + 煙尾前 40%** | ease-out 環 + 1.42× 煙正是本輪賣點；主形已收、煙仍在的幀最有「殘響」 | 2 層 LOD 下無煙尾可賣 |
| **4** | **詞綴精英入場或被集火中** | 玩法支柱上鏡（裂殖／磁滯／疾閃環與標記） | 精英死光後只剩小兵潮＝故事弱 |
| **5** | **升級卡剛滑入 0.3–0.8s** 或 **結算 count-up 中段** | 產品感／Meta；作次要商店圖 | 勿當唯一主圖（缺戰鬥身份） |

| 禁止 | 原因 |
|------|------|

| 開局前 20s 空曠清怪 | 系統未齊（契約已選但無精英／Boss／進化張力） |
| 全進化後期「一片 mint／白青」 | V1 已警告 evo 色相塌縮；可當「強度」不宜當「武器語彙」圖 |

| **C2 小隊關係線** | 五人常糊成一坨；治療金柱／護衛位偶發，截圖難表現「小隊」 | 像單機自機＋召喚物，不像五人 Vanguard | 截圖選 **Echo 金柱命中同幀** 或隊形拉開；中期可加極淡隊員線／小隊序號（debug 可關） | **P1 拍攝／P2 系統** |
| **C3 地圖主題身份** | Press 寫三地圖（虛空／農野／餘燼），主戰鬥圖常分不出「在哪」 | 故事無空間；像同一 arena skin | 每圖強制 **1 個主題錨**（農野作物剪影／餘燼裂縫橙／虛空遠景閃電）佔畫面 8–15% | **P1 拍攝** |
| **C4 因果鏈一幀** | 少見「因→果」同框：契約名、詞綴觸發、進化火花、種子 | 有系統無敘事因果 | L3 圖：**契約標題仍可見**的半透明 HUD 或暫停種子條；或結算「本局種子 + 成就解鎖」 | **P1** |
| **C5 高光殘像可引用** | 無「截圖模式」凍結 3 層 VFX／關閉非必要 HUD | 好瞬間一閃即逝；滿場又被 LOD 砍乾 | debug：`screenshot_beauty=1` → 強制 feature 層、凍幀 0.15s、藏搖桿（V1 順位 5 工具化） | **P2 工具** |
| **C6 情緒音畫占位** | 靜幀無法帶 BGM；但商店常配 5s 循環 | 無聲動圖偏「程式預覽」 | 非本監工範圍；標記給音訊／Press | 範圍外 |

#### 故事感最小公式（給拍攝／下輪設計共用）

```
故事感截圖 ≈ 可讀威脅(C1) + 我方錨點(隊長光/小隊) + 一個規則符號(契約|詞綴|進化) + 空間錨(C3)
             + （可選）因果殘響(爆炸煙尾 / Phase2 色變)
```

四缺一 → 容易變成「特效 reel」。  
V1 已補強 **殘響與威脅色溫**；**C1–C4 仍是截圖故事的主缺口**。
